# Admin全局Copilot助手功能规格说明书

## 概述

为GEA EOR SaaS Admin系统设计一个全局AI助手，以悬浮球形式存在，为不同角色的用户提供智能数据分析和查询服务。

## 核心功能

### 1. 悬浮球UI
- **位置**: 右下角固定位置，可拖拽
- **动画**: 平滑展开/收起，打字机效果
- **响应式**: 适配移动端和桌面端
- **主题**: 与现有设计系统保持一致

### 2. 智能对话
- **自然语言查询**: 支持中英文双语
- **上下文感知**: 基于当前页面和用户角色
- **多轮对话**: 支持连续问答
- **错误处理**: 友好的错误提示和重试机制

### 3. 数据访问控制
- **角色权限**: 严格遵循现有RBAC体系
- **数据脱敏**: 敏感信息自动过滤
- **审计日志**: 所有查询操作记录
- **性能优化**: 查询结果缓存和分页

### 4. 文件分析
- **多格式支持**: PDF、图片、Excel、Word
- **AI解析**: 复用现有Manus LLM服务
- **类型识别**: 自动识别发票、合同、文档
- **安全扫描**: 文件上传前安全检查

## 技术架构

### 前端组件
```
CopilotFloatingBall (悬浮球主组件)
├── CopilotChatPanel (聊天面板)
├── ChatMessage (消息组件)
├── FileUploadArea (文件上传)
└── SuggestedActions (建议操作)
```

### 后端服务
```
copilotRouter (tRPC路由)
├── CopilotDataService (数据服务)
├── CopilotFileService (文件服务)
└── CopilotAIService (AI服务)
```

### 数据流
```
用户输入 → 权限验证 → 数据查询 → AI分析 → 响应生成 → 前端展示
    ↓
文件上传 → 安全扫描 → AI解析 → 结果返回 → 前端展示
```

## 安全考虑

### 1. 数据安全
- 所有数据访问通过现有权限体系
- 敏感信息自动脱敏处理
- 查询结果不包含个人身份信息
- 文件上传限制类型和大小

### 2. 系统安全
- AI服务调用频率限制
- 查询结果缓存防止重复请求
- 错误信息不暴露系统细节
- 定期安全审计和日志分析

## 性能优化

### 1. 前端优化
- 组件懒加载减少初始包大小
- 消息虚拟滚动处理大量对话
- 图片懒加载和压缩
- 本地缓存常用数据

### 2. 后端优化
- 数据库查询优化和索引

- 慢查询监控和告警
