# EOR SaaS Admin — RBAC 权限矩阵

> **版本**: v2.1 | **更新日期**: 2026-03-04

本文档定义了系统中 5 个角色在各模块上的操作权限。权限通过 tRPC 中间件在服务端强制执行。

---

## 1. 角色定义

| 角色 | 代码 | 职责范围 |
|------|------|----------|
| **管理员** | `admin` | 全部权限，系统配置，用户管理，角色分配 |
| **客户经理** | `customer_manager` | 客户管理、员工管理、合同管理 |
| **运营经理** | `operations_manager` | 薪酬管理、假期管理、异动管理、员工状态变更 |
| **财务经理** | `finance_manager` | 发票管理、汇率管理、账单主体管理、发票生成 |
| **普通用户** | `user` | 只读查看、提交异动和假期记录 |

## 3. 权限实现机制

系统认证基于 **JWT + bcrypt + HttpOnly Cookie** 方案。权限控制通过 tRPC 中间件链实现，定义在 `server/procedures.ts`：

| 中间件 | 允许角色 | 用于 |
|--------|---------|------|
| `protectedProcedure` | 所有已认证用户 | 基础认证检查 |
| `userProcedure` | 所有已认证用户 | 通用读写操作 |
| `customerManagerProcedure` | `admin`, `customer_manager` | 客户/员工管理 |
| `operationsManagerProcedure` | `admin`, `operations_manager` | 薪酬/运营管理 |
| `financeManagerProcedure` | `admin`, `finance_manager` | 发票/财务管理 |
| `adminProcedure` | `admin` | 系统管理 |

每个 tRPC 端点在定义时选择对应的中间件，未授权请求返回 HTTP 403 Forbidden。

---

## 4. 数据隔离规则

当前系统为**单租户**设计（内部管理后台），所有角色共享同一数据空间。数据隔离通过以下方式实现：

- **角色隔离**: 不同角色只能访问其权限范围内的操作端点
- **状态隔离**: locked 状态的数据不可被任何角色修改
- **审计追踪**: 所有写操作记录到 audit_logs，包含操作人、操作类型、变更详情
