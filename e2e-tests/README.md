# 🌙 GEA EOR SaaS 夜间自动化测试系统

## 📋 系统概述

这是一个完整的端到端自动化测试系统，专门设计用于在您睡觉后自动运行测试，确保系统各模块间的正确集成和数据一致性。

## 🚀 核心功能

### ✅ 自动化测试覆盖
- **销售流程测试**: Lead创建 → 状态流转 → MSA签署 → 客户转换
- **客户入职测试**: 客户创建 → 合同配置 → 休假政策 → 定价设置
- **员工生命周期测试**: 入职 → 合同 → 休假余额 → 工资项目
- **工资处理测试**: 工资运行 → 工资项目 → 提交 → 锁定
- **发票周期测试**: 发票生成 → 状态流转 → 付款 → 贷项通知单
- **权限安全测试**: 角色权限验证 → 门户隔离

### 🔍 智能特性
- **自动错误检测**: 智能分类错误类型（数据库约束、权限、业务规则、状态转换）
- **自动重试机制**: 失败测试自动重试3次
- **详细报告生成**: HTML、JSON格式报告，包含性能指标和业务指标
- **零容忍数据清理**: 严格按照依赖关系清理数据，确保不污染生产环境
- **夜间调度**: 支持cron定时任务，在您睡觉后自动执行

## 🛠️ 使用方法

### 1. 立即运行测试
```bash
# 运行完整的夜间测试套件
./e2e-tests/nightly-test-runner.sh

# 运行自动测试执行器
./e2e-tests/auto-test-runner.sh

# 运行数据清理验证
./e2e-tests/cleanup-validator.sh
```

### 2. 设置夜间自动测试
```bash
# 初始化调度器
./e2e-tests/test-scheduler.sh setup

# 设置每天晚上11:30运行测试
./e2e-tests/test-scheduler.sh schedule 23:30

# 设置每天凌晨2:00运行测试
./e2e-tests/test-scheduler.sh schedule 02:00

# 查看当前调度状态
./e2e-tests/test-scheduler.sh status

# 取消所有定时测试
./e2e-tests/test-scheduler.sh cancel
```

### 3. 生成测试报告
```bash
# 生成详细测试报告
./e2e-tests/report-generator.sh

# 查看最近的测试日志
./e2e-tests/test-scheduler.sh logs 50
```

## 📊 测试报告说明

### 报告类型
1. **HTML报告** (`test-report-*.html`)
   - 交互式Web界面
   - 测试通过率统计
   - 详细的测试步骤
   - 性能指标和业务指标
   - 建议和下一步行动

2. **JSON报告** (`test-report-*.json`)
   - 机器可读格式
   - 完整的测试数据
   - 便于集成到CI/CD系统

3. **数据验证报告** (`data-validation-*.txt`)
   - 数据清理验证结果
   - 数据库完整性检查
   - 业务规则验证

4. **执行摘要** (`execution-summary-*.txt`)
   - 测试执行详细过程
   - 关键验证点结果
   - 系统性能表现

### 关键指标
- **通过率**: 目标 ≥90%
- **执行时间**: 完整测试套件约5-10分钟
- **数据清理**: 100%测试数据清理
- **业务覆盖**: 覆盖所有核心业务流程

```bash
./e2e-tests/test-scheduler.sh schedule 14:00  # 下午2:00
```

## 🚨 重要提醒

### 零容忍数据政策
- **必须清理**: 所有测试数据必须在测试完成后立即清理
- **生产保护**: 严禁在生产数据库上运行测试
- **数据隔离**: 测试数据必须有明确的标识（如包含"Test"前缀）
- **验证检查**: 每次测试后都要执行数据完整性验证

### 错误处理
- **自动重试**: 失败测试会自动重试3次
- **错误分类**: 自动识别错误类型并提供修复建议
- **详细日志**: 所有错误都会被详细记录
- **通知机制**: 测试失败时会生成警告通知
